# Tests Description - PDF Extractext

## Descripción de Tests por Archivo

### test_api_entry.py
Validación de entrada en endpoints de la API:
- Rechazo de valores null en uploads
- Rechazo de paths vacíos
- Validación de tipos de filename
- Validación de extensiones de archivo
- Validación de contenido vacío
- Validación de tamaño máximo de archivo
- Endpoint de extracción de texto

### test_service_entry.py
Validación de parámetros en la capa de servicios:
- Rechazo de valores null en filename y contenido
- Rechazo de paths/filenames vacíos
- Validación de tipos de entrada
- Validación de extensiones de archivo

### test_extractor.py
Extracción de texto desde archivos PDF:
- Extracción básica de contenido
- Manejo de archivos vacíos
- Extracción por rango de páginas
- Manejo de contenido Unicode

### test_file_management.py
Manejo de archivos y casos edge:
- Archivos no encontrados
- Contenido inválido o corrupto
- Casos edge del sistema de archivos
- Caracteres especiales en filenames

---

## Ejecución de Tests

```bash
# Ejecutar todos los tests
uv run pytest tests/unit/ -v

# Ejecutar archivo específico
uv run pytest tests/unit/test_api_entry.py -v
uv run pytest tests/unit/test_extractor.py -v
uv run pytest tests/unit/test_service_entry.py -v
uv run pytest tests/unit/test_file_management.py -v

# Generar fixtures (si es necesario)
uv run python tests/fixtures/generate_fixtures.py
```
