## Principios Aplicados

### Clean Code
- Funciones pequeñas (<20 líneas)
- Nombres descriptivos (verbos para acciones)
- Sin comentarios innecesarios
- Clases solo cuando agrupan tests relacionados (usando clases originales)

### TDD
- Tests independientes
- Un assert por test (ideal)
- Tests como especificación de comportamiento
- Sin tests acoplados a implementación

### KISS (Keep It Simple, Stupid)
- 7 tests nuevos, no más
- Sin over-engineering
- Soluciones directas y simples
